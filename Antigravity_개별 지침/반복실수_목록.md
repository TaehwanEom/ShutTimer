## 1. 거짓/부정확한 보고 [Critical]

| 패턴 | 사례 출처 | 재발 방지 |
|------|----------|----------|
| 부분 완료를 전체 PASS로 보고 | moda N-4, C-9 | 완료 보고 시 전달문 항목 전체 vs 실제 수정 1:1 대조표 필수 |
| 확인 안 한 항목을 PASS로 기록 | moda 다수 | 확인 안 했으면 "미확인"으로 기록. 추측으로 PASS 금지 |
| 코드 존재 = 기능 동작으로 착각 | moda C-9 | grep으로 코드 있다고 "검증 완료" 금지. 실행 확인 후에만 PASS |
| 미완료를 언급 안 해서 완료로 착각하게 만드는 것 | moda D-6 | 미완료는 반드시 명시. 의도적 생략 = 거짓 보고 |
| 추측으로 에러 원인 보고 | moda 6-3 | 코드를 직접 읽고 흐름 확인 후 보고. 모르면 "확인 중" |

## 7. 공통 규칙 (전 창 적용)

1. **"완료" 보고 전 전달문 항목 전체 1:1 대조.** 빠뜨리면 완료가 아니다.
2. **거짓 보고 금지.** 안 했으면 안 했다고 써라. 부분 성공을 전체 PASS로 보고 금지.
3. **"확인/검토" 지시 = 확인만 하고 보고.** 실행은 별도 승인 후에만.
4. **추측으로 보고하지 마라.** 모르면 "확인 중". 코드를 직접 읽은 후에 보고.
5. **작업 지시 없이 임의 행동 금지.** 세션 복원 시에도 유저 지시 대기.
6. **각 창은 자기 역할만.** 선넘으면 즉시 중단.
